# Yash Poly Plast — Data Collection Checklist
*Please fill in the following details to complete your Knowledge Base. Fields marked **[Client]** require information only you can provide. Fields marked **[Internal]** can be assigned by the QMS consultant.*

---

## 1. Maintenance Plan (`maintenance_plan.json`)

These fields repeat for each of the 44 maintenance tasks (MP001–MP044):

- [ ] **[Client]** Estimated duration (hours) for each maintenance task — how long does each task typically take?
- [ ] **[Client]** Last done date for each task — when was it last performed? (or confirm "Never done / not tracked yet")
- [ ] **[Client]** Next due date for each task — or confirm we should calculate from today's date
- [ ] **[Internal]** Record form code — to be assigned once maintenance forms are drafted (e.g. `YPP_FRM_MNT-01`)

---

## 2. Breakdown Codes (`breakdown_codes.json`)

- [ ] **[Client]** Average downtime (hours) for each of the 12 failure modes — based on past experience. If not tracked, confirm "data not available"

---

## 3. Standards Cross-Reference (`standards.json`)

- [ ] **[Internal]** Key document codes — to be assigned as QMS documents are drafted (procedures, policies, forms). No client input needed; consultant will populate once document list is finalised.
- [ ] **[Client / Internal]** Implementation status review — default is "Planned" for all clauses. Client to confirm which clauses are already partially or fully implemented so status can be updated to "Partially Implemented" or "Implemented".

---

## 4. Legal Register (`legal_register.json`)

- [ ] **[Client]** Compliance status for each regulation — e.g. "Compliant", "Non-compliant", "In Progress", "Not Applicable"
- [ ] **[Client]** Next review date for each regulation — typically 1 year from last review
- [ ] **[Client]** Confirm whether BIS licence (ISI mark) is held for any product — for LR005
- [ ] **[Client]** Confirm EPR registration status under Plastic Waste Management Rules — for LR004
- [ ] **[Client]** Confirm Fire NOC status and expiry date — for LR012

---

## 5. Equipment (`equipment.json`)

- [ ] **[Client]** Serial numbers / asset numbers for EQ001–EQ012
- [ ] **[Client]** Purchase / installation dates for each machine
- [ ] **[Client]** Current calibration certificate numbers and expiry dates for EQ005–EQ012 (instruments)
- [ ] **[Client]** Name of external calibration agency used

---

## 6. Personnel (`personnel.json`)

- [ ] **[Client]** Full names and designations for all staff (if any entries show only designation placeholders)
- [ ] **[Client]** Date of joining for each employee
- [ ] **[Client]** Educational qualifications for each employee

---

## 7. Spares (`spares.json`)

- [ ] **[Client]** Current stock quantity for each spare (SP001–SP010)
- [ ] **[Client]** Reorder level and reorder quantity for each spare
- [ ] **[Client]** Preferred supplier for each spare item
- [ ] **[Client]** Unit price for each spare

---

## 8. Suppliers (`suppliers.json`)

- [ ] **[Client]** Supplier evaluation scores / approved supplier status confirmation
- [ ] **[Client]** Contact details (phone, email) for each supplier
- [ ] **[Client]** Any supplier quality agreements or certificates on file

---

## 9. Customers (`customers.json`)

- [ ] **[Client]** Key contact name and designation at each customer
- [ ] **[Client]** Credit terms and payment history (for commercial records, not QMS — optional)
- [ ] **[Client]** Any customer-specific quality requirements or approved supplier registrations

---

## 10. Products (`products.json`)

- [ ] **[Client]** Actual weight (grams) for each finished product SKU
- [ ] **[Client]** Colour variants available for each product
- [ ] **[Client]** Customer-specific drawing numbers or specifications, if different from internal specs

---

## 11. General / Cross-Cutting

- [ ] **[Client]** Company letterhead details: registered address, GSTIN, CIN, factory address if different
- [ ] **[Client]** Name and signature authority of Management Representative (MR) for document sign-off
- [ ] **[Client]** Date of last management review meeting (for 9.3 records)
- [ ] **[Client]** Date of last internal audit, if any conducted
- [ ] **[Internal]** All document codes (YPP_ prefix) — to be assigned by consultant on document creation
